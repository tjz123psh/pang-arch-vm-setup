# VM Audit Follow-up 2026-07-01

This records how the VM audit findings were handled in this repository.

## Fixed In Installer

- DMS vs mako notification conflict: `mako.service` is disabled because DMS owns notifications in this profile.
- DMS niri adapter: `dms-shell-niri` is installed after the upstream DMS installer.
- fcitx5 environment: `GTK_IM_MODULE`, `QT_IM_MODULE`, and `XMODIFIERS` are now installed through `environment.d`.
- Locale: `en_US.UTF-8` and `zh_CN.UTF-8` are generated if missing.
- Neovim dependencies: `tree-sitter-cli` is now in the official package list.
- Neovide: now in the official package list because the synced nvim config has Neovide-specific settings.
- Fish default shell: installer switches the user shell to fish when needed.
- Rustup fish residue: `fish/conf.d/rustup.fish` is no longer synced, and stale copies are removed if rustup is not installed.
- Nautilus double-click handling: a local `nvim.desktop` wrapper launches `kitty -e nvim`, and directory MIME defaults are set to Nautilus.
- Paru review file manager: switched from fragile `fzf` preview flags to `yazi`.
- `paru-ui` package count: AUR indexing now downloads the official AUR package-name list instead of depending only on local paru completion cache.
- VM essentials: `open-vm-tools`, `python-pip`, user directory creation, and `paru` `SkipReview` are now handled.

## Optional By Design

- WeChat, QQ, Chrome, and Obsidian are not part of the minimal VM base. Install them with `./install.sh --with-apps`.
- App configs, login state, browser profiles, chat data, sync databases, and caches remain excluded.

## Not Installer Bugs

- DMS Chinese UI is missing upstream.
- `paru-ui` first full AUR index build needs network access; if refresh fails it falls back to cached data.
- WeChat bwrap file paste under Wayland can paste file paths instead of file data.
- `~/.face` is cosmetic and can be created manually if desired.
