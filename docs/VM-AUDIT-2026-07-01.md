# VM 审计处理记录 2026-07-01

这份文档记录这次虚拟机安装审计里发现的问题，以及仓库里对应的处理结果。

## 已在安装脚本中处理

- DMS 和 mako 通知冲突：本仓库不安装 mako；如果目标系统已有 `mako.service`，脚本会禁用并 mask。通知由 DMS 接管。
- DMS niri 适配：默认执行等价于官方安装器的 release installer 流程，但通过 GitHub `releases/latest` 跳转解析 tag，避免 `api.github.com` 匿名限流；官方 release installer 失败时回退到 Arch 官方 `dms-shell-niri` 包；DMS 模块排在基础用户配置之后，避免外部下载失败截断 Rime、fish、dotfiles 等部署；同时显式安装 `matugen`、`cava`、`power-profiles-daemon`、`qt6-multimedia`、`qt6ct`、`wtype`、`cups-pk-helper`、`kimageformats` 等常用依赖。
- DMS 权限和服务：用户会加入 `input` 组；`dms.service` 会启用，并在已有图形会话时立即启动，否则下次登录图形会话时启动。
- fcitx5 配置：按当前系统的 Wayland 配置保持简洁，不再强塞 `GTK_IM_MODULE` 到 niri、`environment.d` 或 `~/.profile`；避免 GTK/Wayland 前端反复弹出输入法警告。
- Rime 输入法：默认选中雾凇拼音 `rime_ice`，使用万象 zh-hans 语法；旧的 luna-pinyin 自定义配置会被清理；安装校验会确认最终 `schema_list` 只有 `rime_ice`。
- Locale：确保生成 `en_US.UTF-8` 和 `zh_CN.UTF-8`。
- Neovim 依赖：`tree-sitter-cli` 已加入官方仓库包列表。
- Neovim 主题：关闭 DMS 的 Neovim matugen 模板，因为它会生成 `lua/plugins/dankcolors.lua`，拉取 `RRethy/base16-nvim`，并覆盖 Catppuccin。脚本会清理旧的 `dankcolors.lua` 和 `base16-nvim`。
- Neovide：已加入默认安装，因为当前 Neovim 配置包含 Neovide 专用设置。
- 开发环境：默认包含 `nodejs`、`npm`、`go`、`python`、`python-pip`、`jdk-openjdk`、`clang` 和 `rust`；Arch 官方 `rust` 包提供 `cargo` 和 `rustfmt`，避免 Rust 只作为 AUR 构建依赖残留成孤儿包。
- Fish 默认 shell：脚本会在需要时把用户 shell 切换为 fish。
- Rustup fish 残留：不再同步 `fish/conf.d/rustup.fish`；如果系统没装 rustup，但这个文件还在，会自动清理。
- Nautilus 双击文本文件：本地 `nvim.desktop` 会用 `kitty -e nvim` 打开文本文件；目录 MIME 默认交给 Nautilus。
- Paru 审查流程：`paru.conf` 开启 `SkipReview`；`paru-ui` 和安装脚本都会使用 `paru -S --skipreview`，不再触发文件管理器审查钩子。
- `paru-ui` 包数量：AUR 索引改为下载官方 `packages.gz`，不再只依赖本地 paru completion 缓存。
- VM 基础项：`open-vm-tools`、`python-pip`、用户目录创建、paru `SkipReview` 都已处理。
- 桌面资产：会部署 `~/Pictures/头像`、`~/Pictures/图像`、`~/.face` 和 SDDM Pixie 登录主题。
- OCR/PDF：默认安装 `tesseract`、中文/英文识别数据、方向识别数据、`poppler` 和 `poppler-glib`。

## 个人 VM 范围

- 微信 AppImage、QQ、Chrome、Obsidian 默认安装，因为这个仓库只服务 Pang 自己的 VM；微信包使用 `wechat-appimage` 对齐当前主系统，不使用 `wechat-universal-bwrap`，并会在旧 VM 上移除已经安装的 bwrap 版本。
- 应用登录状态、浏览器 profile、聊天记录、同步数据库和缓存仍然排除在仓库外。

## 不是安装脚本问题

- DMS 中文界面缺失是上游问题。
- `paru-ui` 第一次刷新完整 AUR 索引需要网络；失败时会回退缓存或只显示官方仓库索引，不再调用可能触发 sudo 的 paru 同步流程。
- `wechat-universal-bwrap` 的 Nautilus 文件粘贴问题不再作为默认安装路径处理，因为默认微信包已改为 `wechat-appimage`。
- `~/.face` 只是头像显示用途，缺失不影响系统功能。
